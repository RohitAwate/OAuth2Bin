Authorization Code

Fixed Parameters

Authorization URL
{{.AuthCodeCnfg.AuthURL}}
Access Token URL
{{.AuthCodeCnfg.TokenURL}}
Client ID
{{.AuthCodeCnfg.ClientID}}
Client Secret
{{.AuthCodeCnfg.ClientSecret}}

Request Parameters

response_type=coderequired
Indicates that the application is requesting for an authorization grant token.
client_id={{.AuthCodeCnfg.ClientID}}required
Your client ID.
redirect_uri=...optional
Redirects the user-agent to this address when the authorization is complete.
scope=...optional
Feel free to show your creativity here as the server ignores this.

Process

  • Fetch an authorization grant by sending a GET request to the authorization URL.
    Refer the example request below.
  • Authorization grant is valid for 10 minutes from the time of issue.
  • Use this authorization token to send a POST request to the access token URL.
    Refer the example request below.
  • Access token is valid for 1 hour from the time of issue.
  • Use the access token to make requests to the dummy endpoints.

Example Authorization Grant Request

GET {{.AuthCodeCnfg.AuthURL}}?response_type=code&client_id={{.AuthCodeCnfg.ClientID}}&redirect_uri=https://my-redirect-uri.com&scope=hello-world

Example Access Token Request

POST {{.AuthCodeCnfg.TokenURL}}
Body: grant_type=authorization_code&code=MyAuthGrant&client_id={{.AuthCodeCnfg.ClientID}}&redirect_uri=https://my-redirect-uri.com

Process

  • Open the authorization URL with the required parameters encoded in the URL.
    Refer the example request below.
  • If the user authorizes, an access token is issued.
  • It is valid for 1 hour from the time of issue.
  • Use it to make requests to the dummy endpoints.
